КОНТЕКСТ ПРОЕКТА

Разрабатывается кроссплатформенное мобильное приложение для спортивного ориентирования. Пользователь фотографирует бумажную карту ориентирования, приложение:
1. распознаёт карту (классификация местности по стандарту ISOM: лес/поляна/чаща/тропы/вода/скалы/зоны вне соревнования) и дистанцию (КП — фиолетовые кружки, треугольник старта, двойной круг финиша, номера, соединительные линии курса);
2. строит "стоимостную поверхность" (cost surface) — скорость бега по каждому типу местности;
3. находит оптимальный маршрут между последовательными КП (least-cost path).

Ключевое требование ко всему, что относится к самой дистанции (поиск маршрута, отображение карты): должно работать полностью офлайн, без связи — приложение используется в лесу. Распознавание карты можно (не обязательно) делать с доступом к сети, если пользователь готовит карту заранее дома.

Второй жёсткий принцип: автоматическое распознавание никогда не будет стопроцентно точным. С первой же версии UI должен поддерживать ручную коррекцию (добавить/подвинуть/удалить КП, перекрасить область местности) — без этого одна ошибка распознавания делает маршрут в лесу бесполезным.

АРХИТЕКТУРА

Камера → preprocessing (коррекция перспективы + выравнивание по линиям магнитного севера, которые есть на любой карте ориентирования с равным шагом — удобный якорь для выравнивания и определения масштаба)
→ CV/ML-распознавание (цветовая сегментация ISOM + детекция КП/номеров)
→ ручная коррекция (обязательный UI-слой)
→ векторная карта (GeoJSON-подобная структура: полигоны классов местности + точки КП)
→ cost-grid (скорость бега по клетке)
→ поиск маршрута (Dijkstra/A* на MVP-этапе, впоследствии Fast Marching Method для более реалистичных маршрутов без сеточных артефактов)
→ отрисовка маршрута + офлайн-хранение + опционально live GPS-трекинг.

ЗАФИКСИРОВАННЫЙ СТЕК (не предлагай альтернативы без явного запроса — решение уже принято)

- Клиент: Flutter (Dart), кроссплатформенно iOS/Android.
- Тяжёлые вычисления (CV, path-finding): Rust через flutter_rust_bridge — общий нативно-быстрый код для обеих платформ.
- Классическая CV: OpenCV (через FFI/Rust-биндинги) — коррекция перспективы, HSV-пороги, Hough-детекция кружков, скелетизация линий.
- ML-инференс на устройстве: TensorFlow Lite / ONNX Runtime Mobile.
- Обучение моделей (офлайн, не на телефоне): PyTorch → экспорт в TFLite/ONNX.
- Распознавание номеров КП: Google ML Kit Text Recognition + при необходимости своя лёгкая CNN под шрифт карт.
- Поиск маршрута: прототип логики — Python (scikit-image route_through_array / scikit-fmm), продовая реализация — Rust.
- Внутренний формат карты: GeoJSON-подобная структура; предусмотреть прямой импорт OCAD (.ocd) / OpenOrienteeringMapper (.omap) как обходной путь без CV.
- Офлайн-хранение: SQLite/Drift во Flutter.
- Бэкенд (опционально, не для MVP): Python + FastAPI, только для облачного варианта тяжёлой модели и синхронизации карт.

ФАЗЫ ПРОЕКТА (для ориентира, не обязательно проговаривать все на каждой сессии)

0. Python-прототип CV-пайплайна и path-finding (доказать связь "сегментация → cost-grid → путь" работает).
1. MVP на Flutter: перспективная коррекция, классическая CV-сегментация, автодетекция дистанции (Hough circles на фиолетовом слое), ручная коррекция, Dijkstra/A*.
2. Устойчивость распознавания номеров и картинки (блики/тени/сгибы).
3. ML-сегментация местности (U-Net/DeepLab-lite) вместо/поверх цветовых порогов.
4. Fast Marching Method, учёт склона по горизонталям, live GPS-трекинг.
5. Полировка, тестирование на устройствах, релиз.
